# Standard Operating Procedure Writer

Turns a process description into a clear, step-by-step SOP that anyone can follow — even on their first day.

## What this skill does

Takes a rough process description (or even a brain dump) and structures it into a proper SOP with context, steps, exceptions, and ownership.

## How to use

Describe the process — how it works, who does it, what tools are involved. Say "write an SOP for this process."

## Output format

---

**[PROCESS NAME] — Standard Operating Procedure**

**Version:** 1.0
**Owner:** [Role, not name]
**Last updated:** [Date]
**Applies to:** [Who needs to follow this]

---

**Purpose**
Why this process exists. What goes wrong without it. One paragraph.

**Scope**
When to use this SOP. When NOT to use it.

**Prerequisites**
- Access to [tool/system]
- Completion of [prior step/process]
- [Other requirements]

---

**Steps**

1. **[Step name]**
   - What to do: [specific action]
   - How to do it: [details, where to click, what to enter]
   - Expected outcome: [what you should see/have when done]
   - If something goes wrong: [troubleshooting]

2. **[Step name]**
   [repeat structure]

---

**Exceptions and edge cases**
- If [condition]: [what to do instead]
- If [condition]: [escalate to who]

**Quality checks**
How to verify the process was completed correctly.

**Related processes**
Links or references to related SOPs.

---

## Principles

- Write for someone who has never done this before
- One action per step — no "and then also"
- Name the tool, don't describe it ("in Salesforce" not "in the CRM")
- Include screenshots or video links where steps are complex
- If a step takes more than 5 minutes, it might be its own SOP
